# Launch Playbook

This document is for the first public launch window of Spendline.

## Launch Goal

- 50 active developers
- validate the core pain point
- optimize for installs and feedback, not revenue

## MVP Positioning

Spendline helps developers understand where LLM spend is going, request by request, before the bill becomes a surprise.

## What To Launch With

- hosted web app
- hosted API
- working API key flow
- production smoke test passing
- at least one documented install path for Python and JavaScript users
- clear onboarding path for the first request

## Soft Launch First

Start with a controlled launch before broad distribution:

1. onboard yourself as a real user
2. invite 3-5 trusted developers
3. make sure each user can:
   - sign in
   - generate a key
   - send one tracked request
   - see the request in the dashboard
4. collect friction points

## Public Launch Channels

Use technical content as the main acquisition loop:

- Hacker News
- r/LocalLLaMA
- r/MachineLearning
- LangChain Discord

Suggested article angles:

- Why my LangChain agent cost $400 last week and how I fixed it
- How to track OpenAI spend per feature in production
- How to monitor Claude and GPT costs in one dashboard
- How to track LLM request cost without waiting for the monthly bill
- DeepSeek is cheap, but what is it actually costing you

## Launch Checklist

- Railway healthy
- Vercel healthy
- auth healthy
- API key flow healthy
- ingest healthy
- overview healthy
- request log healthy
- alerts healthy
- smoke test green
- support contact available

## What To Watch After Launch

- install to first-request conversion
- number of generated API keys
- first-request success rate
- alert email delivery
- user feedback on onboarding

## What Not To Add Yet

- teams
- billing
- forecasting
- enterprise controls
- heavy reporting features

The launch goal is signal, not scope expansion.
